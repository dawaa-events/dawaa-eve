# DAWAA Urgent Supabase Booking Link Fix

## المشكلة
المناسبة القديمة كانت ترسل عادي، لكن المناسبة الجديدة لا ترسل، مع رجوع مشاكل الحذف والمزامنة.

## السبب
المناسبة الجديدة كانت تأخذ ID محلي من المتصفح وليس UUID من Supabase.
لذلك الضيوف ينحفظون بدون booking_id صحيح، فتختلط المزامنة والحذف والإرسال.

## الإصلاح
- إضافة API جديد: api/bookings-sync.js
- أي مناسبة جديدة يتم حفظها أولًا في Supabase وتأخذ UUID حقيقي.
- عند حفظ الضيوف، النظام يتأكد أن المناسبة مربوطة في Supabase قبل الحفظ.
- عند تحميل الضيوف، يتم تحميل ضيوف المناسبة الحالية فقط عبر bookingId.
- يمنع اختلاط ضيوف المناسبات القديمة والجديدة.
- الحذف والإرسال يعملان على نفس booking_id الصحيح.

## ارفعي كل الملفات داخل الباتش، خصوصًا:
- app.js
- api/_lib/supabase.js
- api/guests-sync.js
- api/bookings-sync.js

## بعد الرفع
1. افتحي الموقع.
2. أنشئي مناسبة جديدة.
3. تأكدي أنها تنحفظ وتظهر.
4. اختاريها.
5. ارفعي الضيوف.
6. اضغطي تحديث البيانات.
7. جربي إرسال لنفسك أولًا.

## مهم
إذا كان عندك مناسبة جديدة أُنشئت قبل هذا الباتش وفيها مشاكل، الأفضل:
- احذفيها
- أنشئيها من جديد بعد رفع هذا الباتش
حتى تأخذ UUID صحيح من Supabase من البداية.
