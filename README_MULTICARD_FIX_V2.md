# DAWAA Multi-Card RSVP Fix V2

هذا الإصلاح يعالج سبب وصول رسالة التأكيد مباشرة بدل قائمة اختيار عدد البطاقات.

## السبب
عند إعادة استخدام نفس رقم الهاتف في أكثر من تجربة، كان النظام يجد الضيف القديم في Supabase ويحدث `pending_count` فقط، لكنه لا يحدث `cards_count`.
لذلك كان الـ Webhook يرى الضيف كأنه يملك بطاقة واحدة، فيرسل تأكيد مباشر.

## الملفات التي يتم استبدالها فقط

- `api/_lib/supabase.js`
- `api/_lib/meta.js`
- `api/webhook/meta.js`

## ماذا يصلح؟

- عند إرسال دعوة جديدة لنفس الرقم، يتم تحديث:
  - `cards_count`
  - `pending_count`
  - `guest_name`
  - `phone_number`
  - `rsvp_status`

- عند ضغط "أرغب في الحضور":
  - إذا `cards_count > 1` ترسل قائمة اختيار عدد البطاقات.
  - إذا `cards_count = 1` يتم التأكيد مباشرة.

## لم يتم تعديل التصميم
لم يتم لمس:
- `index.html`
- `styles.css`
- `app.js`
- `assets/`
- `supabase.sql`
- `vercel.json`

## اختبار مهم
بعد الرفع:
1. أرسلي دعوة لنفس رقم الاختبار ولكن cards_count = 3.
2. افتحي Supabase → guests وتأكدي أن cards_count أصبح 3.
3. اضغطي "أرغب في الحضور".
4. يجب أن تصلك قائمة اختيار عدد البطاقات وليس رسالة التأكيد مباشرة.
